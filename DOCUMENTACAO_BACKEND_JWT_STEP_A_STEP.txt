DOCUMENTACAO DO BACKEND - JavaScript + JWT + MySQL + DAOs

1) Visao geral
- Backend: Node.js + Express em JavaScript CommonJS.
- Banco: MySQL.
- Acesso ao banco: mysql2/promise com camada DAO em back-end/src/daos.
- Autenticacao: JWT.
- Porta padrao: http://localhost:3000.
- Base da API: http://localhost:3000/api.

2) Como o backend esta organizado
- back-end/src/server.js
  Sobe o servidor HTTP.

- back-end/src/app.js
  Configura CORS, JSON, rotas publicas, middleware JWT global, rotas protegidas, 404 e error middleware.

- back-end/src/routes
  Define metodos HTTP e paths.

- back-end/src/controllers
  Valida entrada com Zod e chama services.

- back-end/src/services
  Regras de negocio, permissao operacional e composicao dos dados.

- back-end/src/daos
  SQL direto no MySQL, mapeamento snake_case/camelCase e consultas com relacionamentos.

- back-end/src/config
  env.js le .env; database.js cria pool MySQL.

- back-end/database/schema.sql
  DDL do banco corridapro com tabelas em portugues: usuarios, equipes, pilotos, carros, pistas, corridas, temporadas, etapas_temporada, voltas_etapa, produtos, pedidos e itens_pedido.

- back-end/database/seed.js
  Cria o database configurado em DB_NAME, recria tabelas e popula dados iniciais alinhados ao front-end.

3) Fluxo de inicializacao
1. server.js importa app.js.
2. app.js registra /api/auth antes do JWT.
3. POST /api/auth/login fica publico.
4. app.js aplica authMiddleware em /api.
5. Todas as demais rotas /api/* exigem Authorization: Bearer TOKEN.
6. Rotas protegidas chamam controllers, services e DAOs.
7. errorMiddleware padroniza erros em JSON.

4) Variaveis de ambiente
Arquivo: back-end/.env

DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=root
DB_NAME=corridapro
DB_CONNECTION_LIMIT=10

PORT=3000
JWT_SECRET="troque_essa_chave"

5) Como preparar o banco
Entre na pasta back-end:

npm install
npm run db:setup

Atencao: npm run db:setup recria as tabelas do banco configurado em DB_NAME.

6) Como iniciar a API
Modo desenvolvimento:

npm run dev

Modo normal:

npm start

7) Login JWT
Endpoint publico:

POST /api/auth/login

Body:

{
  "username": "admin",
  "password": "123456"
}

Resposta:

{
  "token": "...",
  "user": {
    "id": 1,
    "username": "admin",
    "role": "admin",
    "teamId": null,
    "driverId": null
  }
}

Usuarios do seed:
- admin / 123456 / admin
- equipe / 123456 / team
- corredor / 123456 / driver

8) authMiddleware
Arquivo: back-end/src/middlewares/authMiddleware.js

Passos:
1. Le o header Authorization.
2. Exige formato Bearer TOKEN.
3. Valida token com JWT_SECRET.
4. Injeta request.user com id, username, role, teamId e driverId.
5. Se faltar token: 401 "Token JWT nao informado."
6. Se token for invalido/expirado: 401 "Token JWT invalido ou expirado."

9) Permissoes principais
- admin: gerencia cadastros, produtos, corridas e pedidos.
- team: cria corridas, edita corridas da propria equipe, edita pilotos da propria equipe e gerencia pistas.
- driver: consulta dados e faz pedidos na loja.

Observacoes moldadas pelo front-end:
- Team pode criar, editar e excluir pistas porque a interface libera "canManageTracks".
- Team nao exclui corridas porque a interface usa "canDeleteRace: false".
- Frete da loja: gratis a partir de 500; senao 24.90, igual ao front-end.

11) Contrato usado pelo front-end
O front-end chama back-end/front-end/assets/js/api.js com:
- API_BASE_URL = http://localhost:3000/api
- Authorization: Bearer TOKEN salvo em localStorage

Campos importantes mantidos para o front:
- driver.team.name
- driver.cars[0].code ou driver.cars[0].model
- car.team.name
- car.driver.name
- car.packageName
- track.recordLapMs
- race.driver.name
- race.team.name
- race.track.name
- race.car.model
- product.price como number
- order.subtotal, order.shipping e order.total como number
